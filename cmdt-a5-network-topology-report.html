This lab implements a dual-firewall, dual-core-MLSW enterprise HQ campus with redundant distribution-layer 3560 switches (DIS1–DIS5 + DIS6), HSRP gateway redundancy per VLAN, OSPF Area 0 throughout, centralised DHCP, and a DMZ server segment. Internet connectivity via Gate-HQ (serial WAN) with NAT overload and an IPSec crypto map stub.

Redundancy Mechanisms
Dual FWHQ-FW1 + HQ-FW2 each take one uplink from Gate-HQ (10.1.250/251 /30 P2P). FW_LINK interconnects them at Gi1/5.
Dual MLSWHQ-MLSW1 and HQ-MLSW2 each connect to all firewalls and all distribution switches. EtherChannel Po1 (LACP) between them.
HSRPEach VLAN pair is served by two distribution switches with HSRP groups matching the VLAN ID (group 10, 20, 30, 70, 80, 90). Shared VIP is the .1 address of each /24.
DHCP HADual DHCP servers (DHCP-1 / DHCP-2) with identical pools. Both listed in ip helper-address on every SVI — first-response wins.
NATPAT overload on Gate-HQ Serial0/1/0. Internal subnets 10.1.0.0/16 and 10.200.200.0/27 permitted.
STPPVST on all switches. PortFast + BPDUGuard on all access ports. No root-bridge election manipulation captured.
