Network Overview

This document describes the logical network architecture extracted from CMDT A5.pkt. The topology models a mid-to-large enterprise campus with a headquarters (HQ) site connected to a simulated Internet cloud via a serial WAN link, using a redundant dual-firewall perimeter design. The internal campus follows a classic three-tier hierarchical model (Core/Distribution/Access), with VLAN segmentation, HSRP-based gateway redundancy, and centralized DHCP/DNS services in a dedicated DMZ segment.

Architecture Summary

The HQ campus is built around a two-tier core implemented as two 3560-24PS multilayer switches (HQ-MLSW1 and HQ-MLSW2) that form an EtherChannel trunk (Port-channel 1, Fa0/23–24) for redundancy. Each core switch uplinks to one of the two ASA 5506-X firewalls (FW1 and FW2), which in turn connect to the Gate-HQ router (the WAN gateway). This creates an active-active dual-path perimeter.

Below the core layer, five pairs of 3560-24PS distribution switches (DIS1–DIS5 / DIS6 equivalent on the second stack) handle inter-VLAN routing with HSRP, each serving a floor or building segment. Access-layer 2960-24TT switches (Sw1, Sw2, AC3, AC4, AC5 and the DMZ switch) provide port-level VLAN assignment and port-security.

OSPF — Key Design Notes
  • Single Area 0: All OSPF participants are in the backbone area. No inter-area or external LSAs except the default route originated by Gate-HQ.
  • Redistribute static: Gate-HQ redistributes its static routes into OSPF (process 15) with redistribute static subnets, ensuring internal routers learn the default route via Type-5 LSA.
  • Default-information originate: Gate-HQ also uses default-information originate — this pushes a Type-5 default LSA into OSPF so all internal devices can reach the Internet.
  • OSPF on ASA firewalls: Both FW1 and FW2 run OSPFv2 process 15, advertising their internal and external /30 links. This is notable — the ASAs participate in the routing domain, not just as policy enforcement points.
  • Dual-path to Internet: Two paths exist from the campus to the Internet: Core1→FW1→Gate-HQ and Core2→FW2→Gate-HQ. OSPF equal-cost multipathing applies for traffic with identical metrics.
  • Router3 anomaly: Router3 advertises 10.1.20.0/24 and 10.1.80.0/24 — the same subnets as DIS3/DIS4. This appears to be a routing inconsistency in the original file (sub-interface for VLAN80 on Gi0/1.80).

Security Observation
ACL 2 permits management only from 192.168.10.0/24. However, this subnet does not appear anywhere in the network's IP plan — it is not a configured VLAN or interface in the topology. This means VTY access is effectively blocked from all current hosts, which may be intentional (out-of-band management network) or a misconfiguration.

EtherChannel — Core Redundancy Link

HQ-MLSW1 Fa0/23 & Fa0/24 and HQ-MLSW2 Fa0/23 & Fa0/24 form a LACP EtherChannel (Port-channel 1) — MLSW1 is configured as active, MLSW2 as passive. Mode: 802.1Q trunk. This provides both redundancy and bandwidth aggregation between the two core switches.

Recommendations
  • Replace Type-7 password encryption with enable secret (MD5/SHA) on all devices.
  • Verify the management subnet 192.168.10.0/24 is reachable or update ACL 2 to a valid OOB network.
  • Complete or remove the IPSec crypto map on Gate-HQ (define ACL 150 or remove the incomplete map).
  • Restrict the ASA OUTSIDE ACL source addresses for CAPWAP/LWAPP to known WLC peer IPs only.
  • Consider adding HTTPS (TCP/443) to the FW ACL if web services require TLS.
